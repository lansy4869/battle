# 实车使用.md

> 工作区按当前机器写为 `~/slash_ws`。如果车上实际仓库是 `~/jetson_ws`，把下面所有 `~/slash_ws` 替换成 `~/jetson_ws` 即可。  
> 核心原则：先验证底盘、雷达、手柄接管，再上安全壳，最后上学生策略。手柄右扳机接管/停的用法保持不变。

## 0. 上车前准备

### 0.1 编译所有实车相关包

```bash
cd ~/slash_ws
colcon build --symlink-install --packages-select \
  f1tenth_stack livox_ros_driver2 pointcloud_to_laserscan \
  linefit_ground_segmentation linefit_ground_segmentation_ros \
  roboracer_china_2025 slash_safety multilayer_scan obstacle_tracker slash_distill
source install/setup.bash
```

每开一个新终端都先执行：

```bash
cd ~/slash_ws
source install/setup.bash
```

### 0.2 准备学生模型

如果已经有训练好的 ckpt，导出 ONNX：

```bash
cd ~/slash_ws/src/slash_distill
python3 -m slash_distill.export_onnx \
  --ckpt ckpt_dagger/student_best.pt \
  --out ckpt/student.onnx
```

如果暂时没有模型，也可以上车验证整条链路：`race_student.launch.py model_path:=...` 留空时，`student_policy_node` 会全程走反应式兜底，仍然输出 `/drive_raw` 并经过 `ggv_shield`。

TensorRT 主要用于时延基准，可在 Jetson 上跑：

```bash
cd ~/slash_ws/src/slash_distill
bash slash_distill/build_trt.sh ckpt/student.onnx ckpt/student.engine
```

部署节点当前直接加载 `.onnx`，`backend:=trt` 表示优先使用 onnxruntime 的 TensorRT/CUDA provider；没有对应 provider 时会降级或走反应式兜底。

## 1. 首次上电检查

终端 1 启动底盘、雷达、分割、2D scan，但先不启动规划器：

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch roboracer_china_2025 race_reactive_bringup.launch.py run_planner:=false
```

另开终端检查关键话题：

```bash
ros2 topic list | grep -E '^/(livox/lidar|scan|odom|drive|teleop|sensors/imu)'
ros2 topic hz /livox/lidar
ros2 topic hz /scan
ros2 topic hz /odom
```

必须确认：

- 手柄在线，能抢占 `/drive`。
- `/livox/lidar` 有频率。
- `/scan` 有频率。
- `/odom` 有频率。
- 架空车轮测试时，松开手柄接管后车不会自走。

## 2. 推荐首跑：反应式 + g-g-v 安全壳

保持终端 1 的底座在运行。终端 2 启动低速安全壳链路：

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch slash_safety race_with_shield.launch.py \
  scan_topic:=/scan \
  speed_scale:=0.4 \
  max_speed:=1.0 \
  max_steer:=0.34 \
  shield_v_max:=1.0 \
  shield_ay_safety_factor:=0.6
```

链路是：

```text
/scan -> battle_fast2 -> /drive_raw -> ggv_shield -> /drive -> ackermann_mux -> vesc
```

检查输出：

```bash
ros2 topic hz /drive_raw
ros2 topic hz /drive
ros2 topic echo /ggv_shield/v_cap --once
```

确认后，空旷场地低速按住手柄右扳机放行。第一次只做直线、缓弯、静态障碍避让，不追速度。

## 3. 验证 W3 多层感知

保持底座运行，先单独启动多层 scan：

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch multilayer_scan multilayer_scan.launch.py
```

检查：

```bash
ros2 topic hz /scan_3d
ros2 topic hz /perception/scan_layer_low
ros2 topic hz /perception/scan_layer_body
ros2 topic hz /perception/scan_layer_high
```

RViz：

```bash
rviz2
```

Fixed Frame 设 `livox_frame`，添加 4 个 `LaserScan`：

- `/scan_3d`
- `/perception/scan_layer_low`
- `/perception/scan_layer_body`
- `/perception/scan_layer_high`

现场确认：

- 地面不要进 low/body，避免满地假墙。
- 车体高度障碍应进 body 或 low。
- 悬空物只进 high，默认不应阻挡 `/scan_3d`。

如果地面误入，改 `src/slash_perception/multilayer_scan/config/multilayer_scan.yaml` 里的 `band_floor/band_ceil`，重启 launch。

## 4. 验证 W4 动态障碍跟踪

保持底座运行，终端 2 启动 W4 演示链：

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch obstacle_tracker race_w4_demo.launch.py speed_scale:=0.4
```

链路是：

```text
/livox/lidar -> multilayer_scan -> /scan_3d
                         -> obstacle_tracker -> /perception/dynamic_obstacle + /perception/opponent_state
/scan_3d + dynamic flag -> battle_fast2 -> /drive_raw -> ggv_shield -> /drive
```

检查：

```bash
ros2 topic hz /scan_3d
ros2 topic echo /perception/dynamic_obstacle
ros2 topic echo /perception/opponent_state
ros2 topic hz /drive_raw
ros2 topic hz /drive
```

RViz 加 `MarkerArray`，订阅：

```text
/perception/obstacle_markers
```

验证动作：

- 静止纸箱在车前：marker 应为静态，速度接近 0，`dynamic_obstacle=false`。
- 推动纸箱横穿或靠近：marker 应变动态，`dynamic_obstacle=true`，`opponent_state` 里的速度/接近率变化合理。

## 5. 正式学生策略实车启动

### 5.1 启动底座

终端 1：

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch roboracer_china_2025 race_reactive_bringup.launch.py run_planner:=false
```

### 5.2 启动学生 + W3 + W4 + W2

终端 2，先用 ONNX 后端。注意：`race_student.launch.py` 里的护盾默认读取
`src/slash_safety/config/ggv_shield.yaml`，当前 `v_max: 2.0`。学生首跑如果想严格限到
`1.0 m/s`，先临时把该文件里的 `v_max` 改成 `1.0`，稳定后再恢复到 `2.0`。

```bash
cd ~/slash_ws
source install/setup.bash
ros2 launch slash_distill race_student.launch.py \
  model_path:=/home/jetson/slash_ws/src/slash_distill/ckpt/student.onnx \
  backend:=onnx
```

如果还没有模型，先跑兜底链路：

```bash
ros2 launch slash_distill race_student.launch.py model_path:=''
```

链路是：

```text
/livox/lidar -> multilayer_scan -> /scan_3d
/scan_3d + /odom -> obstacle_tracker -> /perception/opponent_state
/scan_3d + /odom + opponent_state -> student_policy_node -> /drive_raw
/drive_raw -> ggv_shield -> /drive -> ackermann_mux -> vesc
```

### 5.3 放车前检查

另开终端：

```bash
ros2 topic hz /scan_3d
ros2 topic hz /perception/opponent_state
ros2 topic hz /drive_raw
ros2 topic hz /drive
ros2 topic echo /drive_raw --once
ros2 topic echo /drive --once
```

看终端 2 日志：

- 出现 `(onnx)`：学生模型在发指令。
- 出现 `(FALLBACK)`：当前帧走反应式兜底。
- 提示 `未配置 model_path` 或 `onnxruntime 载入失败`：模型后端不可用，但节点会继续兜底。

### 5.4 低速实车步骤

1. 车架空，确认 `/drive_raw` 与 `/drive` 有输出，手柄能抢占。
2. 落地，空旷赛道，按住手柄右扳机放行 1-2 秒后松开，确认刹停/接管正常。
3. 单车低速跑半圈，观察是否贴边、蛇形、误刹。
4. 加静态纸箱，确认避让。
5. 推动纸箱做动态障碍，确认 `opponent_state` 和避让/减速行为。
6. 确认稳定后再逐步提高 `ggv_shield.yaml` 的 `v_max` 或 `ay_safety_factor`，不要一次改大。

### 7.2 感知消融素材

录同一个场景：

- 单层 `/scan` 对低矮障碍/悬空物的反应。
- 多层 `/scan_3d` 与 `/perception/scan_layer_*` 对同一障碍的分层结果。
- 静态障碍与被推动动态障碍下的 `/perception/dynamic_obstacle` 和 marker 颜色变化。

这些素材对应 W8 里“3D 多层感知只能实车/定性评估”的部分。

## 8. 常见问题排查

### `/drive_raw` 有，`/drive` 没有

检查安全壳：

```bash
ros2 node list | grep ggv
ros2 topic echo /odom --once
ros2 topic echo /drive_raw --once
```

如果 `/odom` 没有，`ggv_shield` 会因为里程计超时不正常输出。

### `/drive` 有，但车不动

检查 mux 和手柄：

```bash
ros2 topic list | grep -E '^/(drive|teleop|ackermann_cmd|commands/motor/speed)'
ros2 topic echo /teleop --once
```

确认手柄右扳机放行方式正确，且 `/teleop` 没有持续覆盖 navigation。

### `/scan_3d` 没有

检查 MID-360：

```bash
ros2 topic hz /livox/lidar
ros2 node list | grep multilayer
```

如果 `/livox/lidar` 没有，先回到底座/雷达启动排查。

### 学生一直 `FALLBACK`

可能原因：

- `model_path` 路径不对。
- 没装 `onnxruntime`。
- ONNX 的输入维度与当前 `scan_topics` 层数不一致。
- 学生输出和反应式速度差超过 `fallback_speed_jump`，被安全兜底接管。

先用空模型兜底确认整车链路：

```bash
ros2 launch slash_distill race_student.launch.py model_path:=''
```

再检查 ONNX：

```bash
ls -lh /home/jetson/slash_ws/src/slash_distill/ckpt/student.onnx
ls -lh /home/jetson/slash_ws/src/slash_distill/ckpt/student_meta.npz
python3 -c "import onnxruntime as ort; print(ort.get_available_providers())"
```

### 静止障碍被误判动态

调 `src/slash_perception/obstacle_tracker/config/obstacle_tracker.yaml`：

- 误报多：增大 `dynamic_speed_thresh`，或增大 `min_age_for_dynamic`。
- 速度抖：减小 `vel_ema_alpha`。
- 必须保持 `use_ego_compensation: true`。

## 9. 最小推荐实车顺序

1. `race_reactive_bringup.launch.py run_planner:=false`，确认雷达/scan/odom/手柄。
2. `race_with_shield.launch.py speed_scale:=0.4 max_speed:=1.0 shield_v_max:=1.0`，低速跑通安全壳。
3. `multilayer_scan.launch.py`，RViz 调好多层高度带。
4. `race_w4_demo.launch.py speed_scale:=0.4`，验证动态障碍跟踪。
5. `race_student.launch.py model_path:=''`，验证学生部署链路的反应式兜底。
6. `race_student.launch.py model_path:=.../student.onnx backend:=onnx`，低速上学生。
7. 稳定后再做 DAgger 多轮模型、TensorRT 时延、消融录包与论文素材。
