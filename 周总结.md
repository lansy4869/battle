### 2. 完成 g-g-v 物理安全壳 `slash_safety`

本周新建 `slash_safety` 包，用于在规划器输出和底盘 mux 之间增加物理安全约束层。该模块把上游任意策略输出的 `(转向, 速度)` 投影回本车实测可行域，再发布给 `/drive`。

| 交付物 | 功能 | 完成情况 |
| ---- | ---- | ---- |
| `slash_safety/slash_safety/ggv_shield_node.py` | 实现 `/drive_raw -> /drive` 的 g-g-v 安全护盾节点 | 已完成 |
| `slash_safety/config/ggv_shield.yaml` | 集中配置转向、纵向加速度、横向极限、失效保护等参数 | 已完成 |
| `slash_safety/launch/ggv_shield.launch.py` | 单独启动安全护盾 | 已完成 |
| `slash_safety/launch/race_with_shield.launch.py` | 一条命令串联反应式规划器与安全护盾 | 已完成 |

安全壳主要包含四类约束：

- 转向限幅：`|delta| <= 0.34 rad`，使用已有实测转向范围。
- 过弯限速：根据 `kappa = tan(delta) / L` 计算曲率，并限制 `v <= sqrt(a_y,max * factor / kappa)`。
- 纵向变化率限制：加速上限 `+6.35 m/s^2`，制动上限 `-6.66 m/s^2`。
- 失效安全：上游指令超时后，安全壳在约 `0.3 s` 内平滑刹停并转向回中。

> 当前接线方案：规划器发布 `/drive_raw`，安全壳输出 `/drive`，再进入 `ackermann_mux` 的 navigation 通道。该方案不需要修改 mux，只需要使用规划器已有的 `drive_topic` 参数。

### 3. 完成系统辨识补全包 `slash_sysid`

本周新建 `slash_sysid` 包，用于恢复并扩展底盘系统辨识能力。纵向辨识沿用并复现历史实测结果，横向部分新增绕圆 bag 分析能力，用 VESC IMU 实测横向加速度与有效轴距。

| 交付物 | 功能 | 完成情况 |
| ---- | ---- | ---- |
| `slash_sysid/slash_sysid/chassis_sysid.py` | 移植并扩展底盘系统辨识脚本 | 已完成 |
| `slash_sysid/slash_sysid/circle_commander.py` | 绕圆标定指令发生器，支持恒转角、缓升速、自动停车 | 已完成 |
| `slash_sysid/launch/circle_test.launch.py` | 启动绕圆采集流程 | 已完成 |
| `slash_sysid/out/` | 预留系统辨识输出目录 | 已建立 |

系统辨识脚本新增能力包括：

- 支持注册并读取 `vesc_msgs` 类型，解析 `/sensors/imu` 中的 IMU 数据。
- 支持从绕圆 bag 中提取横向加速度 `a_y,max`。
- 支持输出 `lateral` 字段，包括可辨识状态、实测横向加速度、有效轴距、等效摩擦系数等。
- 在 g-g-v 输出中区分横向极限来源：实测值优先，否则标注为 `ASSUMED (mu=1.0)`。

### 4. 复现实测纵向参数并完成验证

本周使用已有真实 bag 对扩展后的系统辨识脚本进行了验证，确认旧有纵向结果没有被破坏，同时验证了 IMU 读取链路。

| 验证项 | 结果 | 说明 |
| ---- | ---- | ---- |
| Python 语法检查 | 全部通过 | `slash_safety` 和 `slash_sysid` 的核心 Python 文件均通过 `py_compile` |
| 纵向模型复现 | `K=0.9139`，`tau=0.255 s`，`Td=0 s`，`R^2=0.973` | 与历史 `identified_params.json` 基本一致 |
| 加速度极限复现 | 加速 `+6.35 m/s^2`，制动 `-6.66 m/s^2` | 已写入安全壳默认参数 |
| 转向范围复现 | `[-0.34, 0.34] rad` | 当前仅能标定 servo 到转向角映射 |
| VESC IMU 读取 | 从 `chassis_test` 读取到 5968 帧 IMU | 证明横向辨识数据链路可用 |
| 横向可辨识性检查 | 当前 `identifiable=false` | 现有 bag 近静止，最高约 `0.31 m/s`，不满足横向辨识条件 |

> 结论：W2 软件链路已经具备横向实测能力；现有 bag 由于车速过低，只能验证 IMU 读取与类型注册，不能得出可靠的横向极限。
