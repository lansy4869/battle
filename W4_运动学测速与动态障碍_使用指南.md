## 0. W4 要解决什么

原 `battle_fast2` 用 LiDAR **intensity 均值差 > 5** 判动态障碍。MID-360 的反射率(0–255)与
2D 雷达 intensity **不可比**，上实车不可移植。W4 用纯运动学替换：

```
/scan_3d ──[obstacle_tracker: 聚类→多帧关联→ego补偿测速]──┬─ /perception/dynamic_obstacle (Bool)
                                                          ├─ /perception/opponent_state (Float32MultiArray)
                                                          └─ /perception/obstacle_markers (RViz)
battle_fast2 订阅 /perception/dynamic_obstacle，用它替代 intensity 判据驱动 跟随/超车 决策。
```

**为什么必须做 ego 补偿**：雷达在车体系观测，车一前进，静止障碍的距离也在变小，帧间看起来
“在动”。所以测到的帧间速度要扣除自车运动：`v_ground ≈ v_meas + v_ego + ω×p`（v_ego/ω 来自 `/odom`）。
补偿后静止障碍 ≈ 0、真动态障碍才保留真实速度。

## 2. opponent_state 话题布局

`/perception/opponent_state`（`std_msgs/Float32MultiArray`，全部实车可测，供创新点 4 学生输入）：

| 下标 | 含义 |
| ---- | ---- |
| 0 | valid（1=有前向障碍，0=无） |
| 1 | range（最相关障碍距离 m） |
| 2 | bearing（方位角 rad，正前方=0） |
| 3,4 | x, y（车体系坐标 m） |
| 5,6 | vx, vy（ego 补偿后的地速 m/s） |
| 7 | speed（速度幅值 m/s） |
| 8 | range_rate（接近率 m/s，负=逼近） |
| 9 | is_dynamic（1/0） |

选取规则：**动态障碍优先，其次取最近**。

## 3. 单独跑 + RViz 验证（核心交付）

前置：W1 底座在跑（出 `/livox/lidar` → 地面分割 → `/segmentation/obstacle`、`/odom`）、W3 多层节点在跑（出 `/scan_3d`）。

```bash
# 只跑测速跟踪器（默认吃 /scan_3d；纯 2D 可 scan_topic:=/scan）
ros2 launch obstacle_tracker obstacle_tracker.launch.py
```

RViz 验证：
1. Fixed Frame 设 `livox_frame`，加一个 `MarkerArray` 显示订阅 `/perception/obstacle_markers`。
2. 把一个纸箱/障碍**静止**放车前：方块应为**绿色**、速度箭头≈0、文字速度≈0。
3. **推动**这个障碍（或它自己移动）：方块变**红色**、出现速度箭头、文字显示 `x.xx m/s DYN`。
4. 看 flag：`ros2 topic echo /perception/dynamic_obstacle` 在障碍运动时应为 `true`。
5. 看 `ros2 topic echo /perception/opponent_state` 确认 vx/vy/speed 合理。

## 4. 关键调参（`config/obstacle_tracker.yaml`）

| 参数 | 作用 | 调参建议 |
| ---- | ---- | ---- |
| `dynamic_speed_thresh` | 判动态的速度阈值 | 默认 0.35 m/s；误报多→调高，漏报→调低 |
| `min_age_for_dynamic` | 至少存活几帧才判动态 | 默认 3；抖动误报→调高 |
| `vel_ema_alpha` | 速度平滑 | 默认 0.5；噪声大→调小（更平滑、更滞后） |
| `cluster_gap_m` | 断簇阈值 | 默认 0.30 m |
| `max_cluster_width_m` | 超过视为墙/大结构丢弃 | 默认 1.2 m；想跟更大目标→调大 |
| `assoc_dist_m` | 帧间关联门限 | 默认 0.6 m；目标快/帧率低→调大 |
| `front_fov_deg` | 只在前向扇区找障碍 | 默认 120° |
| `use_ego_compensation` | ego 运动补偿 | **保持 true**；原地测试可临时 false |

## 5. 全栈 W4 演示（W1底座 + W3 + W4 + W2）

```bash
# 终端1：W1 底座，但不起规划器（规划器由演示 launch 接管）
ros2 launch roboracer_china_2025 race_reactive_bringup.launch.py run_planner:=false

# 终端2：W3 多层 + W4 测速 + 反应式(运动学动态判据) + W2 护盾
ros2 launch obstacle_tracker race_w4_demo.launch.py
```

链路：

```
/segmentation/obstacle → multilayer_scan → /scan_3d
    ├─ obstacle_tracker → /perception/dynamic_obstacle (+opponent_state, markers)
    └─ battle_fast2(scan:=/scan_3d, use_external_dynamic:=true) → /drive_raw
          → ggv_shield → /drive → ackermann_mux(navigation)
```

实车演示顺序（单车、低速、手柄随时接管）：
1. 架空/原地：确认 `/scan_3d`、`/perception/dynamic_obstacle`、`/drive_raw`、`/drive` 都有输出。
2. 静态障碍：车低速接近一个静止纸箱，应避让；该障碍在 RViz 里保持绿色（不误判动态）。
3. 单动态障碍：推动纸箱横穿/逼近，障碍变红，规划器进入 `Follow/跟随`（减速）或避让。
4. 关掉 `obstacle_tracker`：`/perception/dynamic_obstacle` 停发后，`battle_fast2` 的 `dynamic_obs`
   退化为 False（不再臆造动态障碍，也绝不回到 intensity）。

## 6. 与 battle_fast2 的接线说明

- `battle_fast2` 新增参数：
  - `use_external_dynamic`（默认 `true`）：用 `/perception/dynamic_obstacle` 作动态判据。
  - `dynamic_flag_topic`（默认 `/perception/dynamic_obstacle`）。
- 原 `DynamicObastcle()`（intensity 版）方法**保留但不再调用**，仅留作历史对照（消融可对比）。
- `use_external_dynamic:=false` 时，`dynamic_obs` 恒为 `False`（无跟踪器场景的安全默认），**不再使用 intensity**。
