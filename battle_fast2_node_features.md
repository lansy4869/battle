# `battle_fast2_node.py` 新增功能详细解析

与旧版本（复件）相比，最新的 `battle_fast2_node.py` 实现了从“简单反应式逻辑”向“高级行为状态规划与风险管理”的跨越。主要引入了以下三大核心新机制和一系列调试接口。

## 1. 基于有限状态机 (FSM) 的高级行为决策

在旧版中，超车和跟车仅靠简单的布尔值 `self.Follow` 和 `self.chaoche` 通过雷达空间大小进行硬切。新版引入了完整的五状态 FSM，使车辆的超车行为更平滑且符合逻辑。

* **新增状态**：
  * `BEHAVIOR_CRUISE` (巡航): 赛道前方开阔，沿 Raceline 正常行驶。
  * `BEHAVIOR_FOLLOW` (跟随): 前方遇阻且不具备超车条件，降低速度上限（`follow_speed_cap`）跟随。
  * `BEHAVIOR_OVERTAKE_PREPARE` (超车准备): 检测到雷达某一侧存在足够缺口，锁定该侧开始准备超越，但尚未执行大幅度转向。
  * `BEHAVIOR_OVERTAKE_EXECUTE` (超车执行): 确认缺口安全，将局部目标航向强制附加一个偏置角（`overtake_side_bias_deg`），引导车辆主动向空旷一侧并线。
  * `BEHAVIOR_RETURN_TO_RACELINE` (回归主线): 成功超越后，平滑过渡回到全局 Raceline，防止猛打方向。

* **状态切换逻辑 (`_update_behavior_state`)**：
  通过综合分析前方空隙 (`front_clearance`)、障碍物距离、相对速度 (`rel_speed`) 以及保持时间 (`hold_s`) 来严谨地控制状态转移。

## 2. 时序障碍物追踪 (Temporal Obstacle Tracking)

旧版本的障碍物检测是“单帧”的，容易受雷达噪声影响产生闪烁。新版引入了针对动态障碍物的时序追踪队列。

* **历史缓存 (`deque`)**：维护一个长度为 `obs_history_size` 的历史队列，记录障碍物的中心点和距离。
* **相对属性计算**：
  * **相对速度 (`rel_speed`)**：利用相邻帧的时间差 (`dt`) 和距离差，计算出障碍物的相对径向速度。从而判断前车是在远离（负值）还是逼近（正值）。
  * **横向偏移率**：计算中心角度的变化率，识别障碍物是否在横向移动。
* **动态属性判定**：要求目标必须被连续追踪 `obs_min_track_frames` 帧以上，并且存在明显的相对速度或横向移动时，才会被认定为真实的“动态障碍物”。

## 3. 风险感知的模型预测控制 (Risk-aware MPC)

这一部分是本节点最关键的升级。旧版 MPC 主要在“跟踪参考轨迹”和“控制不要太剧烈”之间找平衡；新版则把当前激光雷达数据也塞进代价函数里，让控制器在算最优转角时同时考虑“能不能走”和“会不会撞”。

### 3.1 它到底在控制什么

这里的 MPC 不是连续优化求解器，而是一个有限时域的网格搜索：

- 先枚举一组 `steer`
- 再枚举一组 `accel`
- 对每一对候选控制，向前滚动 `horizon` 步
- 计算总代价
- 取代价最低的那组控制

所以它本质上是“带风险项的离散 MPC rollout”，不是纯启发式，也不是严格求解器。

### 3.2 风险项从哪里来

风险评估的输入主要是：

- `front_state['dis_90']`：前向 180 个角度上的距离数据
- `state['v']`：当前车速
- `ref`：局部参考轨迹
- `speed_upper_bound`：当前行为状态允许的最高速度

风险相关参数是：

- `risk_enable`
- `risk_safe_distance`
- `risk_weight`
- `risk_collision_penalty`

### 3.3 预测轨迹怎么生成

在 `rollout_cost()` 里，控制量在整个预测窗口内视为常量，然后用简化的自行车模型滚动：

- `x += v * cos(yaw) * dt`
- `y += v * sin(yaw) * dt`
- `yaw += v / wheelbase * tan(steer) * dt`
- `v = clip(v + accel * dt, mpc_min_speed, speed_upper_bound)`

这意味着它不是在做复杂的轨迹规划，而是在比较“这一组固定转角和加速度，往前推会变成什么样”。

### 3.4 代价函数由什么组成

总代价不是只看风险，而是四部分叠加：

- 轨迹跟踪误差：`ex, ey, eyaw, ev`
- 控制代价：`steer^2, accel^2`
- 控制平滑项：`(steer - last_mpc_steer)^2`
- 风险项：障碍物安全余量

也就是先保证大方向跟得上赛道，再限制控制别太猛，最后把安全性补进去。

### 3.5 安全余量怎么计算

每个预测步 `k`，系统会把预测点 `(x, y)` 转成一个相对车体的角度：

- `ang = atan2(y, x)`
- 再把这个角度映射到 `dis_90` 的索引

然后取出该方向上的激光距离 `obs_dis`，并计算：

- `pred_dis = sqrt(x^2 + y^2)`
- `margin = obs_dis - pred_dis`

这个 `margin` 表示：

- `margin > 0`：预测点还在障碍物前面，有余量
- `margin = 0`：刚好贴边
- `margin < 0`：模型认为这组控制会穿进障碍物区域

### 3.6 风险惩罚怎么加

代码里分两档处理：

1. `margin < 0`
   - 直接加一个很大的惩罚 `risk_collision_penalty`
   - 目的不是“轻微提醒”，而是把这类候选控制尽量踢出最优解

2. `0 <= margin < risk_safe_distance`
   - 加二次惩罚
   - 惩罚大小随 `risk_weight` 增强
   - 还乘了一个阶段权重 `1 + 0.35 * k`

这个阶段权重很重要，表示越往预测后段，风险代价越高，避免车辆为了当前一步好看，后面却把自己逼进死角。

### 3.7 `min_margin` 的意义

`rollout_cost()` 会在整个 horizon 内记录最小余量 `min_margin`，`solve_mpc()` 会把它一起返回。

这个值不是直接参与最终控制的唯一依据，但非常有用：

- 可以看这次 MPC 选出来的动作有多“贴边”
- 可以在日志里追踪风险变化
- 可以配合 `risk_min_margin_m` 话题做调参

### 3.8 它的实际效果

有了风险项以后，MPC 的偏好会变成：

- 赛道跟踪误差差一点没关系，但别离障碍物太近
- 如果某个候选转向会更贴边，它会被压低优先级
- 如果某个候选已经明显会撞上去，几乎一定不会被选中

所以它不是“硬碰硬地避障”，而是“在追线和安全之间做带权折中”。

### 3.9 这套方法的边界

它仍然有明显假设：

- 只看当前一帧激光，不是完整地图
- 用的是 2D 极坐标距离，不是真正的占据栅格
- 风险是软约束，不是绝对安全保证
- 如果激光被遮挡、噪声大，`margin` 也会受影响

换句话说，这套 MPC 更适合“比赛中快速反应 + 保守规避”，不适合做强全局最优规划。

## 4. 全面的 ROS 2 诊断与可观测性体系

新版极大地丰富了可供 `rqt_plot` 或 `rviz2` 观测的话题以及可动态调整的超参数。

* **新增微调参数**：例如 `risk_weight`（风险权重）、`fsm_execute_opening_threshold`（执行超车的空隙阈值）、`obs_min_track_frames` 等 20 余个参数，便于在实车调试时精确控制。
* **新增 Debug 话题**：
  * `/battle_fast2/behavior_state` (当前 FSM 状态)
  * `/battle_fast2/obs_distance_m` (障碍物距离)
  * `/battle_fast2/obs_rel_speed_mps` (相对速度)
  * `/battle_fast2/risk_min_margin_m` (MPC评估出的最小安全距离)
  配合定时器周期打印，为分析因撞车或异常减速导致的问题提供了充足的数据支撑。
