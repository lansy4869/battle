## 0. 一句话

MID-360 是三维雷达，但现有管线只取**一层**薄切片压成 2D `/scan`，把高度信息全丢了。
本周加一个 **多层投影节点**：把点云按**多个高度带**投影成多条 scan（低矮 / 车体 / 悬空），
融合成给反应式规划器的 `/scan_3d`——能区分「真挡路的车体高度障碍」与「可从下方通过的悬空物」。

**输入与现有 /scan 同源**：默认订阅 `/livox/lidar`（原始点云）。本车**当前未跑 linefit**，
现有 `/scan` 也是 `pointcloud_to_laserscan` 直接吃 `/livox/lidar` 生成的，所以多层节点同源、不引入新依赖。
（以后若启用 linefit，可 `input_topic:=/segmentation/obstacle` 并把 low 下界放更低。）

**避地面 & 改进点**：雷达在 base_link 上方 ~0.11m，`livox_frame` 内 z≈0 为雷达平面、地面约 z≈-0.11 以下；
band 下界放在地面之上即可避地面。各阻挡带逐角度取**最近距离**融合，悬空层 `high` **不阻挡** ——
这正是相对单层 `/scan` 的改进：高处门楣/桌沿不再当假墙。band 高度是 RViz 可调起点，**必须上车标定**。

## 2. 单独跑 + RViz 验证（核心交付）

整车 bringup（出 `/livox/lidar`）在跑的前提下：
```bash
ros2 launch multilayer_scan multilayer_scan.launch.py
```
输出话题：
- `/scan_3d` —— 融合后给规划器的 scan；
- `/perception/scan_layer_low`、`_body`、`_high` —— 每层调试 scan。

**RViz 验证步骤**：
1. `rviz2`，Fixed Frame 设 `livox_frame`。
2. 加 4 个 `LaserScan` 显示，分别订阅上面 4 个话题，设不同颜色（如 low=黄、body=绿、high=红、fused=白）。
3. 把已知物体放到车前验证高度分层是否合理：
   - 矮路肩/纸箱 → 应只进 `low`/`body`；
   - 与雷达同高的墙/车 → 进 `body`；
   - 门楣/桌沿等高处 → 只进 `high`（且**不**进 `/scan_3d`）。
4. 确认**地面没有进任何阻挡层**（否则会有满地假墙）。

## 3. 调高度带（关键，按 RViz 现象调）

高度带定义在 `livox_frame` 的 z 轴，单位 m，见 [`config/multilayer_scan.yaml`](src/slash_perception/multilayer_scan/config/multilayer_scan.yaml)：
```yaml
input_topic:   /livox/lidar              # 与本车 /scan 同源（未跑 linefit）
band_names:    ["low",  "body",  "high"]
band_floor:    [-0.08, -0.05,   0.10]
band_ceil:     [-0.05,  0.10,   0.50]
band_blocking: [true,   true,   false]   # high 默认不阻挡（悬空）
```
调参口诀：**地面不进 low/body、真障碍进 low/body、悬空物只进 high**（band 下界须在地面之上）。
- 地面在 `low`/`body` 里出现满地假墙 → 抬高 `low` 的 `band_floor`（如 -0.08 → -0.04）。
- 矮障碍被漏 → 适当降低 `low` 的 `band_floor`（但别碰到地面）。
- 不想要多层、想退回单层 → 只留 `body` 一条带（主干仍成立）。
- 自车体支架成固定假墙 → 调大 `ego_box_x/ego_box_y`。

## 6. 诚实状态 & 与后续的衔接

- **地面分割**：本仓库**有** linefit 包（`linefit_ground_segmentation_ros`），但**本车当前实车启动未使用**，
  `/scan` 直接由 `pointcloud_to_laserscan(/livox/lidar)` 生成。本节点因此默认也吃 `/livox/lidar`，
  靠高度带下边界避地面（不是真正的地面分割）。
  若后续启用 linefit，可 `input_topic:=/segmentation/obstacle` 并把 low 下界放更低；
  更鲁棒的坡道/不平地面处理可在 W5 引入轻量地面估计。
- **新颖性定位**：全网 F1TENTH 竞速基本都是单层 2D；「MID-360 多层结构做反应式竞速决策」是平台差异化（创新点 1）。
- **喂给创新点 4**：这几条 `/perception/scan_layer_*` 正是 **BC 学生网络的 3D 多层输入**——
  W3 在感知层做的事，W6 直接复用为学生观测（多层 scan 堆叠）。
