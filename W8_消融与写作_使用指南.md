## 0. 为什么消融是「可靠性后盾」

即使创新点 4（学生）效果平平，**消融本身就是可发表结论**——
「在一台 3D-LiDAR F1TENTH 上，到底什么对竞速与 sim-to-real 有用」。这保证「无论如何都有结果」。

## 1. 消融矩阵：哪些在仿真做、哪些在实车做

| 维度 | A | B | 指标 | 在哪做 |
| ---- | ---- | ---- | ---- | ---- |
| 车辆参数 | gym 默认(0.33m) | **sysid 接地** | 圈速/完成率/轨迹一致 | **仿真** |
| 安全壳 | 无 g-g-v | **有 g-g-v 投影** | 越界率/平滑度/圈速 | **仿真** |
| 控制后端 | 纯反应式 | **base+BC 学生** | 圈速/完成率/时延/回退率 | **仿真** |
| 训练范式 | BC | **DAgger** | 完成率/协变量漂移 | **仿真** |
| **感知维度** | 单层 2D | **3D 多层** | 低/高障碍区分 | **实车/定性** |
| 学生结构 | （可选）GRU | 无状态 CNN | 时延/TRT 兼容 | 仿真+Jetson |

> ⚠ **感知维度（3D）在仿真内无法度量**：f110_gym 是 2D、无高度。这一行必须用**实车/定性**素材
> （见 §4），与方案文档「对低/高障碍区分正确率」一致——这是诚实表述，不是缺陷。

## 2. 跑仿真消融

```bash
cd ~/slash_ws/src/slash_distill

# 全跑（有 BC 与 DAgger ckpt 时，4 个维度都跑；多地图多 seed 提升统计力）
python3 -m slash_distill.run_ablations \
    --bc-ckpt ckpt/student_best.pt \
    --dagger-ckpt ckpt_dagger/student_best.pt \
    --maps vegas berlin skirk --seeds 0 1 2 --episodes 10 \
    --out ablation/results.csv

# 只跑反应式相关行（车辆参数 / 安全壳；无需 ckpt）
python3 -m slash_distill.run_ablations --maps vegas --episodes 10 --out ablation/results.csv

# smoke-test（已验证）
python3 -m slash_distill.run_ablations --episodes 1 --max-steps 60 --maps vegas --out /tmp/abl.csv
```

CSV 每行：`ablation, variant, map, seed, grounding, shield, <8 个指标>`。

## 3. 出表 + 出图

```bash
python3 -m slash_distill.make_figures --csv ablation/results.csv --out-dir ablation
```

产出：
- `ablation/summary.md`——每个维度的 `(variant × 指标)` 均值表（**始终生成**，直接贴论文）。
- `ablation/ablation_<维度>.png`——对照柱状图（装了 matplotlib 才出；本机已装，已验证）。

核心指标：单圈时间/完成率/碰撞率、sim-to-real（同策略 sim vs 实车低速）、
推理时延（Jetson < 20 ms / 50 Hz）、安全壳触发统计、回退触发率。

## 4. 实车 / 定性素材（感知维度 + sim-to-real）

这些**不能在仿真出**，按 W3/W4 指南在车上录：

- **3D vs 2D 对照镜头**：同一**低矮障碍 / 悬空物**下，单层 `/scan` 漏检、`/scan_3d` 命中（W3 指南 §留存素材）。
- **动态判据对照**：intensity（旧）vs 运动学（W4）在同一动态障碍下的误报/漏报（W4 指南 §7）。
- **sim-to-real 圈速差**：同一策略在「sysid 接地仿真」与「实车低速」的圈速/轨迹差异。
- **部署时延**：`build_trt.sh` 的 trtexec 基准 + 节点 `mean_inference_ms`。

1. 引言：3D MID-360 F1TENTH 的平台差异化 + 部署优先动机。
