## 1. 接地参数从哪来（与车上同一套数字）

`src/slash_distill/config/sysid_params.yaml` 与 `src/slash_safety/config/ggv_shield.yaml` **保持一致**，
避免 sim 与车上漂移：

| 参数 | 值 | 来源 |
| ---- | ---- | ---- |
| `wheelbase` | 0.25 | vesc.yaml 实测 |

| `long_gain_K / long_tau_s / long_delay_s` | 0.9139 / 0.255 / 0 | chassis_sysid 复现 |

`grounding=default` 时则用 `gym_defaults`（lf=0.15875, lr=0.17145 → 0.33m），供 W8「sysid vs 默认」消融对照。

## 2. 采集专家示范（核心交付）

```bash
cd ~/slash_ws/src/slash_distill

# 正式采集（200 episodes，单车，自动域随机化）
python3 -m slash_distill.collect_demos --episodes 200 --out data/demos.npz

# 带对手（产 opponent_state 真值，供学生对手通道）
python3 -m slash_distill.collect_demos --episodes 200 --num-agents 2 --out data/demos_2car.npz

# 指定地图（放入你自己的赛道后）
python3 -m slash_distill.collect_demos --maps /abs/path/track1 /abs/path/track2 --episodes 300

# smoke-test（已验证可跑）
python3 -m slash_distill.collect_demos --episodes 2 --max-steps 60 --out /tmp/demo_smoke.npz
```

输出 `data/demos.npz`：`lidar (N, C, n_beams)`、`scalar (N, S)`、`action (N, 2=steer,speed)`，
外加 meta（`n_layers, n_beams, scalar_dim, steer_max, v_max`）。

> **地图**：默认用 gym 自带 `vegas`（你尚未放图）。放图方式：把 `<track>.png/.yaml` 放好，
> `--maps` 传**绝对路径（不含 .yaml）**。若同目录有 `<track>_centerline.csv`（每行 `x,y[,v]`），
> 专家自动切到 **pure-pursuit**；否则回退**反应式专家**（日志里 `expert=pursuit/reactive` 可见）。

## 3. 观测布局（部署 = 训练同构，定义在 `common/obs.py`）

| 部分 | 内容 | 仿真 | 实车 |
| ---- | ---- | ---- | ---- |
| lidar | n_beams×n_layers 通道，重采样定长、按 range_max 归一化 | 1 层（gym 2D scan） | 3 层（/scan_3d 的 low/body/high） |
| scalar | `ego_speed` 归一化 + `opponent_state[10]` 归一化 | 同 | 同（opponent_state 来自 W4 obstacle_tracker） |

> ⚠ **2D 仿真无高度**：仿真里多层会退化（各层相同）。学生网络层数无关，
> 「3D 多层收益」在实车/定性消融体现（见 W8）。

## 4. 关键调参（`config/distill.yaml` 的 `collect` / `obs`）

| 参数 | 作用 | 建议 |
| ---- | ---- | ---- |
| `collect.n_episodes` | 采集 episode 数 | 200–500；地图多则多 |
| `collect.num_agents` | 1=单车 / 2=带对手 | 想学跟随/避让用 2 |
| `collect.domain_rand.mu_range` | 路面摩擦随机 | [0.8,1.1]，增强迁移 |
| `collect.domain_rand.lidar_noise_std` | lidar 噪声(m) | 0.01 起 |
| `obs.n_beams` | lidar 重采样束数 | 108（默认）；越大网络越大 |
| `obs.use_opponent` | 是否拼 opponent_state | 单车也建议 true（占位 0，结构一致） |

## 6. 诚实状态 & 与后续衔接

  我们用 K/τ/Td 在命令侧逼近，几何/极限直接替换。论文 sim-to-real 章节据此写
  「文献 0.33m / 假设 0.8g vs 本车 0.25m / 实测」的失配。
- **横向极限仍假设**：补绕圆 bag 后改 `sysid_params.yaml` 即真接地。
- **数据直接喂 W6**：`data/demos.npz` 就是 BC 训练输入。
