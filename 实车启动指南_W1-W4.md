# 实车启动指南（W1–W4 改动后）

> 工作区：`~/jetson_ws`（开发机为 `~/下载/studies/slash_ws/jetson_ws`）。
> 本文对照你**原来的 4 条启动命令**，说明改动后怎么启动。安全开关（手柄右扳机）用法不变：
> `/drive` 仍走 `ackermann_mux` 的 navigation 口，手柄 `/teleop` 优先级更高，松开扳机即可接管/停。

```bash

  roboracer_china_2025 slash_safety multilayer_scan obstacle_tracker slash_sysid

```

## 改动对你原命令的影响（先看这个）

你原来的 4 条命令里，**只有第 4 条（规划器）行为变了**，前 3 条（底盘 / 雷达 / p2l）**完全不变**：

| 原命令 | 现在 |
| ---- | ---- |
| `ros2 launch f1tenth_stack bringup_launch.py` | **不变** |
| `ros2 launch livox_ros_driver2 msg_MID360_launch.py` | **不变** |
| `ros2 run pointcloud_to_laserscan ...`（你那条 p2l） | **不变**（W1 基础档要；W3/W4 档可不依赖它） |
| `ros2 launch roboracer_china_2025 battle_fast2.launch.py` | 仍可用，但现在**默认保守速度、不再 print 刷屏、去掉了 intensity 动态判据** |

规划器新增的默认行为：
- `speed_scale=0.6`、`max_speed=2.0`、`max_steer=0.34`：最终速度/转向被保守限幅（上车更安全）。
- `verbose=false`：不再满屏 print（要看调试日志就 `verbose:=true`）。
- `use_external_dynamic=true`：动态障碍判据改用运动学跟踪话题 `/perception/dynamic_obstacle`；
  **若不启动 W4 跟踪器，则恒为「无动态障碍」**（不会再用不可移植的 intensity）。

## 档位一：W1 基础档（最接近你原来的跑法）

终端 1–3（和你原来一模一样）：

```bash
# 终端1：底盘 + 手柄 + mux
ros2 launch f1tenth_stack bringup_launch.py

# 终端2：MID-360 雷达 -> /livox/lidar
ros2 launch livox_ros_driver2 msg_MID360_launch.py

# 终端3：点云 -> /scan（你原来的 p2l 命令，保持不变）
ros2 run pointcloud_to_laserscan pointcloud_to_laserscan_node \
    --ros-args \
    -r cloud_in:=/livox/lidar \
    -r scan:=/scan \
    -p target_frame:=livox_frame \
    -p range_min:=0.1 \
    -p range_max:=20.0 \
    -p transform_tolerance:=0.5
```

终端 4（规划器，现在保守）：

```bash
ros2 launch roboracer_china_2025 battle_fast2.launch.py
# 想更慢起步： speed_scale:=0.4
# 想看调试日志： verbose:=true
ros2 launch roboracer_china_2025 battle_fast2.launch.py speed_scale:=0.4 verbose:=true
```

然后按住手柄右扳机让它跑（和以前一样）。

> 说明：W1 基础档发布到 `/drive`，没有 g-g-v 安全壳。要安全壳/3D/测速请看下面档位。

## 档位二：W1 + W2 安全壳（推荐日常用）

前 3 条命令同上（底盘 / 雷达 / p2l），**第 4 条换成**：

```bash
ros2 launch slash_safety race_with_shield.launch.py
```

它会：`battle_fast2 → /drive_raw → ggv_shield → /drive`，规划器输出先过 g-g-v 物理护盾
（转向限幅 / 过弯限速 / 加减速限制 / 上游超时刹停）再上底盘。手柄接管不受影响。

验证：

```bash
ros2 topic hz /drive_raw      # 规划器原始输出
ros2 topic hz /drive          # 护盾后输出
ros2 topic echo /ggv_shield/v_cap   # 当前过弯限速
```

## 档位三：W1 + W2 + W3 三维多层感知（创新点 1）

把单层 `/scan` 升级成多层 `/scan_3d`（区分低矮 / 车体 / 悬空障碍，悬空不挡路）。

前 2 条命令（底盘 / 雷达）同上。**p2l 这条可以不开**（多层节点直接吃 `/livox/lidar`）。
第 3 条换成一条全栈：

```bash
ros2 launch multilayer_scan race_3d.launch.py
```

链路：`/livox/lidar → multilayer_scan → /scan_3d → battle_fast2 → /drive_raw → ggv_shield → /drive`。

**首次务必先 RViz 标定高度带**（否则可能地面假墙或漏检），见 `W3_三维多层感知_使用指南.md`：

```bash
ros2 launch multilayer_scan multilayer_scan.launch.py   # 只出 /scan_3d 和各层调试 scan
rviz2   # Fixed Frame=livox_frame，加 LaserScan 看 /scan_3d 和 /perception/scan_layer_{low,body,high}
```

> 高度带默认 `low=[-0.08,-0.05] body=[-0.05,0.10] high=[0.10,0.50]` 只是起点，**必须按 RViz 现象调**
> （改 `src/slash_perception/multilayer_scan/config/multilayer_scan.yaml` 后重编 `multilayer_scan`）。

## 档位四：W1+W2+W3+W4 全栈（含运动学测速 / 动态障碍，创新点 2）

在多层感知基础上加 `obstacle_tracker`：用多帧运动学 + ego 运动补偿测速，替换 intensity 判据，
产出 `/perception/dynamic_obstacle` 与 `/perception/opponent_state`。

前 2 条命令（底盘 / 雷达）同上。第 3 条换成：

```bash
ros2 launch obstacle_tracker race_w4_demo.launch.py
```

链路：
```
/livox/lidar → multilayer_scan → /scan_3d
    ├─ obstacle_tracker → /perception/dynamic_obstacle (+opponent_state + RViz markers)
    └─ battle_fast2(scan:=/scan_3d, use_external_dynamic:=true) → /drive_raw → ggv_shield → /drive
```

RViz 看 `/perception/obstacle_markers`：静止障碍=绿、运动障碍=红+速度箭头。
单车演示（推一个纸箱即可）：静态避让 + 单动态障碍跟随/避让。详见 `W4_运动学测速与动态障碍_使用指南.md`。

## 安全须知（每次上车）

1. **先架空/支起车轮**确认 `/drive` 有输出、方向正确，再落地。
2. 手柄常在手，右扳机即安全开关；松开/打到遥控即接管。
3. 场地空旷，首跑用低 `speed_scale`（0.3~0.4），确认稳定再逐步放开。
4. W2 横向极限 `ay_max` 当前仍是假设 `9.81`（绕圆 bag 实测前别声称已标定），`ay_safety_factor` 保持 `0.6`。
5. W3 高度带、W4 速度阈值都需在实车按 RViz/现象标定，别直接信默认值。

## 常见问题

- **规划器不动 / 速度很小**：`speed_scale` 默认 0.6 是保守值，确认后可调高；或档位用了护盾，检查 `/drive_raw` 有没有输出。
- **没有动态障碍反应**：W1/W2/W3 档不含测速器，`dynamic_obs` 恒为 false；要动态判据请用档位四（W4）。
- **`/scan_3d` 满地假墙**：高度带下界太低吃到地面，抬高 `multilayer_scan.yaml` 里 `low` 的 `band_floor` 重编。
- **想看规划器调试输出**：加 `verbose:=true`（默认静默，避免刷屏拖慢实车）。
